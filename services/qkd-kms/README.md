# QKD Key Management System (`qkd-kms`)

A FastAPI sidecar service that wraps the **BB84 quantum key distribution protocol** as a Key Management System for the Quantum Banking System.

---

## ⚠️ Simulation Notice

This service **simulates** BB84 using a classical pseudo-random implementation. It does **not** use a real quantum channel.

| Real QKD | This PoC |
|---|---|
| Fibre-optic quantum channel | In-process Python simulation |
| Single-photon sources & detectors | `random` module |
| Information-theoretic security | Computational security only |
| Privacy amplification & error correction | SHA-256 key derivation |

In production, replace `_simulate_bb84()` in `app/kms.py` with a real QKD hardware SDK call.

---

## Endpoints

### `POST /kms/session/new`
Create a new QKD-derived 256-bit AES key session.

```bash
curl -s -X POST http://localhost:8001/kms/session/new \
  -H "Content-Type: application/json" \
  -d '{"peer": "api-gateway"}' | jq
```

**Response:**
```json
{
  "key_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "key_b64": "base64-encoded-32-byte-key",
  "qber": 0.0312,
  "scheme": "BB84-sim"
}
```

> `qber` is the Quantum Bit Error Rate. In a real channel, a QBER > 11% indicates eavesdropping and key negotiation should be aborted.

---

### `GET /kms/session/{key_id}`
Retrieve an existing session by its key ID.

```bash
curl -s http://localhost:8001/kms/session/3fa85f64-5717-4562-b3fc-2c963f66afa6 | jq
```

**Response:**
```json
{
  "key_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "key_b64": "base64-encoded-32-byte-key",
  "qber": 0.0312,
  "scheme": "BB84-sim",
  "peer": "api-gateway",
  "caveat": "⚠ PoC only: key is re-readable. Real QKD keys must be consumed once."
}
```

Returns `404` if the session does not exist.

---

### `GET /health`

```bash

```

**Response:**
```json

```

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `PORT` | `8001` | Listening port (set via `uvicorn` CLI in Dockerfile) |
| `PYTHONUNBUFFERED` | `1` | Ensures logs are flushed immediately |
| `PYTHONDONTWRITEBYTECODE` | `1` | Prevents `.pyc` file generation |

No additional configuration is required for the PoC. For production, add:

| Variable | Description |
|---|---|
| `REDIS_URL` | Redis connection string for persistent session store |
| `QBER_THRESHOLD` | Maximum acceptable QBER (default: `0.11`) |

## Running Locally

### Prerequisites

- Python 3.11+
- (Optional) a virtual environment

```bash
cd services/qkd-kms

# Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Start the service
uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload
```

The service is available at `http://localhost:8001`.

### Running Tests

```bash
cd services/qkd-kms
pytest tests/ -v
```

## Architecture

```
api-gateway  ──POST /kms/session/new──►  qkd-kms
                                            │
                                            ▼
                                    _simulate_bb84()
                                    (classical BB84 sim)
                                            │
                                            ▼
                                    _derive_key()
                                    (SHA-256 of sifted bits)
                                            │
                                            ▼
                                    KMSStore (in-memory dict)
                                            │
                ◄──── key_id + key_b64 ─────┘

api-gateway  ──GET /kms/session/{id}──►  qkd-kms
                ◄──── key material ────────────
```

The `api-gateway` uses the returned `key_b64` as a 256-bit AES key for encrypting sensitive payloads before persisting them, providing a quantum-resistant (in simulation) layer of key management.
